Summary

It launches several simulations covering all combinations of parameter values as indicated by the user.

Analyze Results Tab

IMPORTANT: The user must browse the desired set of experiments to analyze. The directory needs to be exactly the directory that contains the model_info.json file and the experiment_specs.json files. If any other directory is provided the program can experience unexpected behavior. This tab contains the following:
  • Parameter:run_id
  • Scatter Plot
  • Histogram Plot
  • Line Plots
See the documentation for more details on how to use these options.

Simulation

Set the start and end time of the simulation

Variables

Select the state variables for which a sweep analysis is desired. At least one of the state variables of the model should be selected.

Parameters

Specify for each parameter of the model the desired perturbation type: Fixed, Sweep or None.

Plot

(Optional) Calculate and plot upper bound and lower bound curves, which emerge from the bundle of curves produced by all the combinations of perturbations tested.

Known limitations

IMPORTANT: the number of combinations of parameter values that can be swept without considerable performance cost is approximately 30 combinations. If more are requested, the program might take too long to generate all plots if the option for plotting upper and lower limits is turned on (and might even crash). This is due to the number of compare operations required that grows exponentially whith the number of parameters to find the upper and lower bounds. This issue will be addressed in a following release. Only parameters and variables of type Real are recognized. Renamings like 'type MyType = Real' are not supported either. Arrays of any type are also not yet supported.
